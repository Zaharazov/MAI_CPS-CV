## 1. Выбор начальных условий:

Для выполнения данной лабораторной работы был выбран набор данных [Intel Image Classification](https://www.kaggle.com/datasets/puneet6060/intel-image-classification)

Обоснование выбора: Данный датасет содержит около 25 000 изображений, распределенных по 6 категориям естественных сцен: горы, леса, здания, моря, ледники и улицы. Практическая значимость: Задача автоматической классификации ландшафтов и городских сцен имеет критическое значение для развития систем компьютерного зрения в беспилотных летательных аппаратах и картографических сервисах. Точное определение типа местности позволяет алгоритмам навигации корректировать маршрут или выбирать соответствующие режимы полета/движения в зависимости от окружения

Для оценки эффективности моделей классификации выбраны: **Accuracy** и **F1-score**

**Accuracy** - является стандартной метрикой для задач многоклассовой классификации, показывающей общую долю правильно предсказанных категорий. Поскольку классы в используемом датасете относительно сбалансированы, Accuracy дает наглядное представление об общей производительности модели

**F1-score** - выбрана как более надежный индикатор качества, учитывающий баланс между Precision и Recall для каждого класса. Это критически важно, так как для практических задач одинаково нежелательны как пропуски объектов, так и ложные срабатывания. Использование метрики позволяет убедиться, что модель одинаково хорошо работает на всех типах сцен

## 3. Улучшение бейзлайна:

Были проверены три гипотезы улучшения:

1. **H1 (Аугментации данных):** RandomResizedCrop, RandomHorizontalFlip, ColorJitter - для повышения обобщающей способности моделей
2. **H2 (Подбор learning rate):** уменьшение шага обучения для более стабильной тонкой настройки
3. **H3 (Регуляризация):** переход на оптимизатор AdamW с weight_decay = 1e-4 для предотвращения переобучения и улучшения обобщения

**Обоснование гипотез:**
1. Аугментация данных (H1) была выбрана для искусственного расширения датасета и повышения устойчивости моделей. Поскольку модели трансформеров и глубокие сверточные сети требуют больших объёмов данных, такие трансформации, как зеркальное отражение и случайное кадрирование, заставляют нейросеть выделять устойчивые признаки объектов, а не запоминать конкретные пиксели

2. Подбор шага обучения (H2) важен при использовании предобученных на ImageNet весов. Слишком высокий learning rate может разрушить полезные признаки, полученные на большом датасете. Уменьшение шага позволяет проводить аккуратный fine-tuning

3. Регуляризация через AdamW + weight decay (H3) направлена на борьбу с переобучением. Оптимизатор AdamW корректно отделяет веса от weight decay, что приводит к более простым и обобщающим решениям

**Результаты проверки гипотез:**
| Конфигурация ResNet18 | Accuracy | F1-score | Изменение (Acc) |
| :--- | :---: | :---: | :---: |
| Бейзлайн | 0.9320 | 0.9336 | - |
| Гипотеза 1 (Аугментации) | 0.9307 | 0.9316 | -0.14% |
| Гипотеза 2 (Уменьшенный LR) | 0.9307 | 0.9320 | -0.14% |
| Гипотеза 3 (AdamW) | 0.9350 | 0.9360 | +0.32% |

| Конфигурация ViT_B_16 | Accuracy | F1-score | Изменение (Acc) |
| :--- | :---: | :---: | :---: |
| Бейзлайн | 0.9453 | 0.9466 | - |
| Гипотеза 1 (Аугментации) | 0.9477 | 0.9487 | +0.25% |
| Гипотеза 2 (Уменьшенный LR) | 0.9437 | 0.9448 | -0.17% |
| Гипотеза 3 (AdamW) | 0.9443 | 0.9451 | -0.11% |

**Выводы по улучшению бейзлайна:**

- По результатам экспериментов **лучшим улучшением** для ResNet-18 стала гипотеза H3 (AdamW + weight decay), которая дала прирост +0.32% по Accuracy  
- Для ViT-B/16 наиболее эффективной оказалась гипотеза H1 (аугментации данных) с приростом +0.25%

В качестве **улучшенного бейзлайна** было решено использовать:
- Для ResNet-18 - конфигурацию H3 (AdamW + weight decay)
- Для ViT-B/16 - конфигурацию H1 (аугментации данных)

Эти конфигурации показали наилучший баланс между точностью и обобщающей способностью и будут использоваться в дальнейшем сравнении с собственной имплементацией модели

## 4. Имплементация алгоритма машинного обучения:

Для решения задачи классификации была спроектирована и реализована собственная архитектура сверточной нейронной сети.

К ней были применены те же улучшения, что и для двух готовых моделей:
| Конфигурация MyCNN | Accuracy | F1-score |
| :--- | :---: | :---: |
| **MyCNN (Base)** | 0.6817 | 0.6746 |
| MyCNN + H1 (Аугментации) | 0.6837 | 0.6772 |
| MyCNN + H2 (Уменьшенный LR) | 0.6240 | 0.6115 |
| MyCNN + H3 (AdamW) | 0.6747 | 0.6667 |

1. Аугментация (H1): )
Для модели, которая учится с нуля, аугментация - это единственный способ увидеть больше данных. За 10 эпох сеть успела выучить общие формы объектов (контуры зданий, гор), а не просто запомнить конкретные пиксели. Это дало честный, хоть и небольшой прирост

2. Уменьшенный LR (H2): 
Уменьшение шага обучения замедлило сходимость простой модели, из-за чего она не успела полностью обучиться за 10 эпох

3. AdamW (H3):
Оказал наиболее негативное влияние. Сильная регуляризация через weight decay оказалась слишком жёсткой для небольшой архитектуры MyCNN, существенно ограничив её способность выучивать сложные признаки

**Вывод:**

Проведённые эксперименты показали, что эффективность методов улучшения зависит от типа модели и способа её обучения. Подходы, связанные с регуляризацией и тонкой настройкой оптимизации (AdamW, уменьшенный learning rate), оказались наиболее полезными для предобученных архитектур, где важно сохранить уже извлечённые признаки и аккуратно адаптировать модель к новой задаче

В то же время для моделей, обучаемых с нуля, ключевую роль играет увеличение разнообразия обучающих данных. Аугментации оказываются наиболее эффективным способом повышения качества, тогда как избыточная регуляризация или снижение шага обучения могут ограничивать способность модели к обучению и ухудшать итоговые результаты
